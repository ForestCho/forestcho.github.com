初学MongoDb-windows上安装mongodb

2013-11-24
畅游互联网
mongodb, open-source, 数据库

MongoDB is an open-source document database that provides high performance, high availability, and automatic scaling.MongoDb 名字源自humongous（极大的，巨大无比的），可见是在处理海量数据和高并发性有很大的优势。同MYSQL一样属于开源项目。属于nosql数据库中的一种。其数据以文本的形式存储。用C++编写，现在支持java,c++,php，python等主流常见的编程语言，可运行在Windows,Linux，Mac Os 平台上。

1、从http://www.mongodb.org/downloads下载与你操作系统对应的mongodb，系统是32位的就下载32位的版本。

2、下载完后可解压，然后在D盘新建一个mongodb的目录，再将解压了的文件中的bin目录下的所有文件(.exe,.pdb)拷贝到建好的mongodb里面去。

3、在d:\MongoDB\目录下建一个data的目录用来存产生的数据。

4、接下来是如何配置，进入打开你cmd命令窗口，进入d:\mongodb\目录下，输入mongod –dbpath D:\MongoDB\data，窗口中出现如下的数据就说明配置成功，且成功启动。


当然你也可以这样查看是否启动，在浏览器输入：http://localhost:27017/，27017为默认的服务监听端口，28017为网络监听端口，You are trying to access MongoDB on the native driver port. For http diagnostic access, add 1000 to the port number

用dom4j读取和更新xml文件

2013-10-20
畅游互联网
dom4j, 操作xml, 配置文件

平时项目中，涉及到xml解析的操作很多，解析http请求，读取配置文件。解析XML的方法很多，有dom，sax等，作为一个初学者，我发现还是DOM4J比较适合我的使用习惯。他是一种开源的xml API，提供大量的API方法（不用我们自己去写），简单高效。
configure.java

public class configure {
    public String username=null;
    public String password=null;
    public void setUsername(String username){
        this.username=username;
    }
    public void setPassword(String password){
        this.password=password;
    }
    public String getUsername(){
        return this.username;
    }
    public String getPassword(){
        return this.password;
    }
}

readFile.java 操作xml数据的类，有读取xml和更新后保存xml的方法。 

import java.io.File;
import java.io.FileWriter;
import java.io.IOException;
import java.io.PrintWriter;
import java.io.Writer;
import java.util.List;
import java.util.Vector; 
import org.dom4j.Document;
import org.dom4j.DocumentException;
import org.dom4j.DocumentHelper;
import org.dom4j.Element;
import org.dom4j.io.OutputFormat; 
import org.dom4j.io.SAXReader;
import org.dom4j.io.XMLWriter;

public class readFile {
    public String filePath = "WebRoot\\WEB-INF\\test.xml";
    public String filePath1 = "WebRoot\\WEB-INF\\test1.xml";
    Vector confVec = null;

    public void writeXml() {
        try {
            Document doc =  DocumentHelper.createDocument(); 
            Element root = doc.addElement("configures"); 
            for (int i = 0; i < confVec.size(); i++) {
                configure conf = (configure) confVec.get(i);
                Element confE = root.addElement("configure").addAttribute("id", Integer.toString(i));

                Element usernameE = confE.addElement("username");
                usernameE.setText(conf.getUsername());

                Element passwordE = confE.addElement("password");
                passwordE.setText(conf.getPassword()); 
            }
               OutputFormat format = new OutputFormat("    ", true);
               Writer out=new FileWriter(filePath1);
               format.setEncoding("gb2312");
               // 可以把System.out改为你要的流。
               XMLWriter xmlWriter = new XMLWriter(out, format);
               xmlWriter.write(doc);
               xmlWriter.close();

        } catch (IOException e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
    }

    public void readXml() {
        try {
            confVec=new Vector();
            Document doc; 
            SAXReader reader=new SAXReader();
            doc = reader.read(new File(filePath));             
            Element root=doc.getRootElement(); 
            List confList=root.elements("configure");
            for (int i = 0; i < confList.size(); i++) {
                Element confE=(Element) confList.get(i);
                configure conf = new configure(); 
                if(i==1){
                    confE.element("username").setText("ccc");//此处进行了update
                }
                String username = confE.element("username").getText();            
                String password = confE.element("password").getText();
                 System.out.println(username+":"+password);
                 conf.setUsername(username);
                 conf.setPassword(password);
                confVec.add(conf);
            }
        }  catch (DocumentException e) {
            // TODO Auto-generated catch block
            e.printStackTrace();
        }
    }
}

apache 配置完成后，不少启动会发现the rquested operation has failed 这个问题，我也是深受其迫害，

原因1:80端口被其他程序占用，一旦出现这个问题首先呢想到的是端口被占用，这时可以C:\Program Files\Apache Software Foundation\Apache2.2\conf（默认安装路径）下将http.conf,的参数Listen 80改为Listen 1234（别的端口），这个比较简单，当然你也可以找出占用这个端口的程序，一般为iis和迅雷，然后关了他们。

原因2：我按照第一部做了后发现没什么用，这并不能说明第一步是错误的，应该还是有其他的错误，绝对多数是loadmodule和phpinidir出现路径的错误，一般是指向的目录不存在，今天网上翻翻有个方法很好使
